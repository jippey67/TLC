In this project, youll use your knowledge of computer vision techniques to build a classifier for images of traffic lights! You'll be given a dataset of traffic light images in which one of three lights is illuminated: red, yellow, or green.

In this notebook, you'll pre-process these images, extract features that will help us distinguish the different types of images, and use those features to classify the traffic light images into three classes: red, yellow, or green. The tasks will be broken down into a few sections:

  4. Classification and visualizing error. Finally, you'll write one function that uses your features to classify any traffic light image. This function will take in an image and output a label. You'll also be given code to determine the accuracy of your classification model.

  5. Evaluate your model. To pass this project, your classifier must be >90% accurate and never classify any red lights as green; it's likely that you'll need to improve the accuracy of your classifier by changing existing features or adding new features. I'd also encourage you to try to get as close to 100% accuracy as possible!

2. Pre-process the Data¶

After loading in each image, you have to standardize the input and output!

Input¶

This means that every input image should be in the same format, of the same size, and so on. We'll be creating features by performing the same analysis on every picture, and for a classification task like this, it's important that similar images create similar features!

Output¶

We also need the output to be a label that is easy to read and easy to compare with other labels. It is good practice to convert categorical data like "red" and "green" to numerical data.

A very common classification output is a 1D list that is the length of the number of classes - three in the case of red, yellow, and green lights - with the values 0 or 1 indicating which class a certain image is. For example, since we have three classes (red, yellow, and green), we can make a list with the order: [red value, yellow value, green value]. In general, order does not matter, we choose the order [red value, yellow value, green value] in this case to reflect the position of each light in descending vertical order.

A red light should have the label: [1, 0, 0]. Yellow should be: [0, 1, 0]. Green should be: [0, 0, 1]. These labels are called one-hot encoded labels.

(Note: one-hot encoding will be especially important when you work with machine learning algorithms).

Standardize the output¶

With each loaded image, we also specify the expected output. For this, we use one-hot encoding.

  • One-hot encode the labels. To do this, create an array of zeros representing each class of traffic light (red, yellow, green), and set the index of the expected class number to 1.

Since we have three classes (red, yellow, and green), we have imposed an order of: [red value, yellow value, green value]. To one-hot encode, say, a yellow light, we would first initialize an array to [0, 0, 0] and change the middle value (the yellow value) to 1: [0, 1, 0].

def standardize(image_list):
    
    # Empty image data array
    standard_list = []

    # Iterate through all the image-label pairs
    for item in image_list:
        image = item[0]
        label = item[1]

        # Standardize the image
        standardized_im = standardize_input(image)

        # One-hot encode the label
        one_hot_label = one_hot_encode(label)    

        # Append the image, and it's one hot encoded label to the full, processed list of image data 
        standard_list.append((standardized_im, one_hot_label))
        
    return standard_list

def create_feature(rgb_image, low_thres):
    
    ## TODO: Convert image to HSV color space
    hsv = cv2.cvtColor(rgb_image, cv2.COLOR_RGB2HSV)
    ## TODO: Create and return a feature value and/or vector
    v = hsv[:,:,2]
    feature = []
    for row in v:
        feature.extend(row)
    
    thres_im = np.copy(v)
    mask = cv2.inRange(thres_im, low_thres, 255)
    
    thres_im[mask == 0] = [0]
    center = thres_im[3:30,10:22]
    #plt.imshow(center,cmap = 'gray')
    feature = []
    for row in center:
        feature.extend(row)
    
    return feature

def find_average_index(rgb_image, low_thres):
    features = create_feature(rgb_image,low_thres)
    #plt.imshow(rgb_image)
    indexes = np.argwhere(np.array(features)>0)
    #print(indexes)
    average_index = np.average(indexes)
    std_dev = np.std(indexes)
    return average_index, std_dev

(QUESTION 1): How do the features you made help you distinguish between the 3 classes of traffic light images?¶

Answer: I used the v-layer of the image in hsv space. The idea behind this choice is that a light should produce high value v-pixels surrounded by much lower values as the traffic light box and the switched off lights are dark in images. As many images also contain light pixels surrounding the non-square traffic light I decided to clip off 3 pixels from the top and bottom and 10 pixels from the left and right side. The remaining v values should provide the location of the pixels for the light that is 'ON'. All v-values are concatenated in a 1-D array. Within this array a search is done, based on:

  • the value being above a certain low_threshold
  • what's the average location of the pixels above this threshold (those should represent the location of the light that is 'ON')
  • IF the average location is below yellow_green (y_g) threshold: predict GREEN above the red_yellow (r_y) threshold: prediction RED
  • ELIF the average location is below the red_yellow threshold: predict YELLOW
  • ELSE: predict GREEN
  • After all predictions are done, the misclassified images are checked upon predicting green on a red light.

For the values of low_threshold, r_y, y_g a grid search is done to find the values with the highest accuracy, where a set that contains red lights classified as green are excluded.

    all_safe=True

(Question 2): After visualizing these misclassifications, what weaknesses do you think your classification algorithm has? Please note at least two.¶

Answer:

1) A high low_threshold, here meaning that only pixels with of above 205 are taken into account, will have difficulty rightly classifying very dark images as no pixels with a value above the threshold are found at all, resulting in a 'NaN' for the average location of the light pixels. The selection criteria were built to predict RED for this situation in order to prevent wrongly giving the GREEN light. Image 3 plotted above is an example of this phenomenon

2) Image 1 is very light over all. The pixels the feature is looking for are all over the place (demonstrated by a high standard deviation in the found locations), resulting in an average prediction of ORANGE.

3) Images that don't fit well in the clipped box that I chose will have pixels surrounding the traffic light dominate the image. Image 2 is an example of that, where the many bright pixels left/above the actual traffic light box cause a RED prediction.
